## 1. 重构目标

当前目标是重写官网 web 端亲属关系工具，解决以下问题：

| 问题 | 目标 |
|---|---|
| 规则混乱 | 建立统一规则表 |
| 页面逻辑过重 | 算法下沉到 utils/kinship |
| 五层限制不稳定 | 用状态机统一控制 |
| 终点关系还能继续点 | 按钮禁用并显示原因 |
| 祖辈规则不完善 | 使用精确路径规则 |
| 姻亲规则容易丢失 | 保留大小舅子、连襟、妯娌等规则 |
| 文件乱码 | 统一 UTF-8 |
| 补丁过多 | 改为完整文件重构 |

## 3. 正式文件结构

第一版重构不要拆太碎，先保持稳定。

~~~text
web/src/utils/kinship/
├─ types.ts
├─ helpers.ts
├─ allowedRelations.ts
├─ terminalRules.ts
├─ resolveKinship.ts
└─ testCases.ts
~~~

后续如果规则继续复杂，再拆：

~~~text
ancestorRules.ts
spouseRules.ts
siblingRules.ts
collateralRules.ts
~~~

## 4. 核心类型设计

### BaseRelation

~~~ts
export type BaseRelation = 'parent' | 'child' | 'sibling' | 'spouse'
~~~

### Gender

~~~ts
export type Gender = 'male' | 'female' | 'unknown'
~~~

### KinshipStep

~~~ts
export interface KinshipStep {
  relation: BaseRelation
  gender: Gender
  birthday?: string
  age?: number | string
}
~~~

### KinshipSelf

~~~ts
export interface KinshipSelf {
  gender: Gender
  birthday?: string
  age?: number | string
}
~~~

### KinshipContext

~~~ts
export interface KinshipContext {
  self: KinshipSelf
  steps: KinshipStep[]
}
~~~

### KinshipResult

~~~ts
export interface KinshipResult {
  title: string
  matched: boolean
  terminal: boolean
  reason: string
  process: string[]
  allowedNextRelations: BaseRelation[]
}
~~~

### RelationOption

~~~ts
export interface RelationOption {
  value: BaseRelation
  label: string
  disabled: boolean
  reason?: string
}
~~~

## 5. 基础输入规则

### 基础关系只允许四类

| 中文 | 内部值 |
|---|---|
| 父母 | parent |
| 子女 | child |
| 兄弟姐妹 | sibling |
| 配偶 | spouse |

### 推导依据

亲属称谓只能根据以下信息推导：

| 信息 | 是否参与推导 |
|---|---|
| 基础关系路径 | 是 |
| 本人性别 | 是 |
| 对方性别 | 是 |
| 本人出生日期 | 是 |
| 对方出生日期 | 是 |
| 本人年龄 | 是 |
| 对方年龄 | 是 |
| 姓名 | 否 |

> 姓名只用于页面展示，不能参与亲属称谓计算。

### 年龄和生日

| 情况 | 处理 |
|---|---|
| 有生日 | 优先用生日判断长幼 |
| 没生日但有年龄 | 用年龄判断长幼 |
| 生日和年龄都没有 | 返回泛称 |
| 生日相同 | 根据性别返回兄弟或姐妹 |
| 年龄相同 | 根据性别返回兄弟或姐妹 |

## 6. 规则优先级

推导时必须按顺序执行，后面的规则不能覆盖前面已经精确命中的结果。

| 优先级 | 规则 |
|---|---|
| 1 | 五层限制 |
| 2 | 终点关系限制 |
| 3 | 精确路径规则 |
| 4 | 祖辈规则 |
| 5 | 姻亲规则 |
| 6 | 堂表规则 |
| 7 | 侄甥规则 |
| 8 | 通用兄弟姐妹规则 |
| 9 | fallback 泛称 |

## 7. 五层限制

最多支持 5 层关系。

### 达到 5 层后

| 项目 | 处理 |
|---|---|
| 添加按钮 | 禁用 |
| 按钮样式 | 变灰 |
| 提示文案 | 最多只能添加 5 层关系 |
| 点击行为 | 显示提示，不允许无反应 |

### 直系路径允许到 5 层

~~~text
父母 -> 父母 -> 父母 -> 父母 -> 父母
子女 -> 子女 -> 子女 -> 子女 -> 子女
~~~

## 8. 选项限制状态机

这是本工具的关键逻辑，不允许再用零散判断打补丁。

### 第一层

| 当前状态 | 允许选择 |
|---|---|
| 空路径 | 父母、子女、兄弟姐妹、配偶 |

### 连续祖辈路径

| 路径状态 | 允许 | 禁止 | 原因 |
|---|---|---|---|
| parent | parent、sibling、spouse | child 谨慎限制 | 避免绕回 |
| parent -> parent | parent、sibling、spouse | child | 避免从祖辈绕回 |
| parent -> parent -> parent | parent、sibling、spouse | child | 避免高层祖辈回跳 |
| parent -> parent -> parent -> parent | parent | child | 第五层不能再选子女 |
| 5 层 parent | 无 | 全部 | 已达 5 层 |

### 连续后代路径

| 路径状态 | 允许 | 禁止 | 原因 |
|---|---|---|---|
| child | child、sibling、spouse | parent 谨慎限制 | 避免绕回 |
| child -> child | child、sibling、spouse | parent | 避免回跳 |
| child -> child -> child | child、sibling、spouse | parent | 避免回跳 |
| child -> child -> child -> child | child | parent | 第五层不能再选父母 |
| 5 层 child | 无 | 全部 | 已达 5 层 |

### 配偶路径

| 路径 | 允许 | 禁止 |
|---|---|---|
| spouse | parent、sibling、child | spouse |
| spouse -> parent | sibling、parent | spouse、child 需谨慎 |
| spouse -> sibling | spouse、child | parent |
| spouse -> sibling -> spouse | 无 | 全部 |
| spouse -> sibling -> child | child、spouse | parent |

### 兄弟姐妹路径

| 路径 | 允许 | 禁止 |
|---|---|---|
| sibling | spouse、child | parent、sibling 谨慎 |
| sibling -> spouse | 无 | 全部 |
| sibling -> child | child、spouse | parent |
| sibling -> child -> spouse | 无 | 全部 |

## 12. 页面交互验收标准

| 验收项 | 要求 |
|---|---|
| 结果区 | 放在页面上方 |
| 推导过程 | 一眼能看到 |
| 添加按钮 | 可用时正常，禁用时变灰 |
| 禁用提示 | 必须显示明确原因 |
| 点击禁用按钮 | 显示提示，不允许没反应 |
| H5 展示 | 表单紧凑，不占太多高度 |
| 中文 | 不能出现乱码 |
| 页面 | 不能空白 |
| 下拉选项 | 必须是中文 |
| 视觉 | 字体和卡片不能过大 |

## 14. 禁止事项

| 禁止项 | 原因 |
|---|---|
| 反复 patch 同一个 Vue 文件 | 容易越改越乱 |
| 创建大量 .bak 文件 | Git 状态混乱 |
| 创建临时 ps1/cjs 后不删除 | 项目污染 |
| PowerShell 大段写中文源码 | 容易乱码 |
| 页面里写复杂算法 | 不可维护 |
| 为 web 工具改 backend/admin/miniprogram | 扩大影响范围 |
| 丢失已确认规则 | 会反复回退 |
| 根据姓名推导称谓 | 逻辑错误 |
