# 亲属关系推导规则整理

本文档用于整理当前项目中的亲属关系推导规则，后续用于优化：

- web/src/views/KinshipTool.vue
- web/src/utils/kinship/resolveKinship.ts
- web/src/utils/kinship/allowedRelations.ts
- web/src/utils/kinship/normalizePath.ts
- web/src/utils/kinship/helpers.ts

当前工具限制：最多 5 层关系。

推导依据：

- 基础关系
- 性别
- 出生日期
- 年龄
- 完整路径

不根据姓名推导。

---

## 1. 基础关系

用户只选择四类基础关系：

- 父母
- 子女
- 兄弟姐妹
- 配偶

其他称谓全部由系统推导。

---

## 2. 年龄和生日规则

生日和年龄有一个有值即可参与判断。

优先级：

1. 双方都有生日，按生日判断。
2. 双方都有年龄，按年龄判断。
3. 一方生日一方年龄，生日换算成年龄后判断。
4. 都没有年龄和生日，只能给泛称。

同龄兄弟姐妹不能显示“兄弟姐妹”，应该根据性别判断：

- 男性同龄：兄弟
- 女性同龄：姐妹

## 16. 当前重点问题

1. 祖辈路径算法必须重构，不能按层数粗暴判断。
2. 母系、外祖母分支不能错误合并。
3. 曾祖父母、高祖父母、外高祖父母、外曾外祖父母要按完整路径生成。
4. 终止关系按钮必须变灰并提示原因。
5. allowedRelations 和 resolveKinship 必须保持一致。
6. 每次重构后必须跑自动自检脚本。
7. 第一层兄弟姐妹同龄时，必须根据性别输出兄弟或姐妹。
8. 生日和年龄任意一个有值即可参与判断。
9. 配偶默认性别需要根据上一层人物自动推导。
10. 配偶的父母需要正确输出岳父、岳母、公公、婆婆。
