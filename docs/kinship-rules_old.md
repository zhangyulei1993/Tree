# 四代内称谓规则表

## 约定

- 视角人统一称为“我”。
- 基础关系只建议存：父母/子女、配偶、兄弟姐妹。
- 称谓不直接入库，由算法根据路径、性别、年龄或排行推导。
- “兄/弟、姐/妹、伯/叔、大/小、二/三/四/五/六……”依赖年龄、出生日期或同辈排行，排行称谓不能写死数量。
- 本表先按常见普通话称谓设计，地域称谓后续可做方言配置。

## 基础字段

| 字段 | 用途 |
| --- | --- |
| gender | 判断父亲/母亲、儿子/女儿、哥哥/姐姐等 |
| birthday 或 birth_year | 判断长幼 |
| birthday 或 birth_year | 同辈排行的主要依据，系统自动计算大哥、二姐、六嫂等 |
| birth_order | 仅作为缺少出生日期时的兜底字段，默认不要求用户手动填写 |
| spouse_id 或 spouse relationship | 判断配偶关系、姻亲关系 |
| parent_child relationship | 推导父母、子女、祖辈、孙辈、伯叔姑舅姨 |
| sibling relationship | 父母不完整时，补充兄弟姐妹关系 |

## 排行称谓生成规则

排行称谓应按数字动态生成，不在规则表里枚举到固定数量。四代、五代、六代甚至更多代时，同辈人数可能很多，算法必须支持“四哥、五姐、六嫂、七弟、八妹”等。

排行默认由系统按年龄自动计算：

1. 优先使用完整生日 `birthday`。
2. 没有完整生日时使用出生年份 `birth_year`。
3. 生日或年份都缺失时，才使用可选的 `birth_order` 兜底。
4. 如果同辈多人年龄无法比较，称谓退化为“哥哥/姐姐/弟弟/妹妹”等非精确排行称谓，或提示资料不完整。

| 排行 | 前缀 |
| --- | --- |
| 1 | 大 |
| 2 | 二 |
| 3 | 三 |
| 4 | 四 |
| 5 | 五 |
| 6 | 六 |
| 7 | 七 |
| 8 | 八 |
| 9 | 九 |
| 10 | 十 |
| 11 | 十一 |
| 12 | 十二 |
| 20 | 二十 |
| 21 | 二十一 |

| 基础称谓 | 生成示例 |
| --- | --- |
| 哥 | 大哥、二哥、三哥、四哥、五哥、六哥、七哥 |
| 姐 | 大姐、二姐、三姐、四姐、五姐、六姐、七姐 |
| 弟 | 二弟、三弟、四弟、五弟、六弟、七弟 |
| 妹 | 二妹、三妹、四妹、五妹、六妹、七妹 |
| 嫂 | 大嫂、二嫂、三嫂、四嫂、五嫂、六嫂 |
| 伯 | 大伯、二伯、三伯、四伯、五伯 |
| 舅 | 大舅、二舅、三舅、四舅、五舅 |
| 姨 | 大姨、二姨、三姨、四姨、五姨 |
| 姑 | 大姑、二姑、三姑、四姑、五姑 |

建议算法提供一个通用函数：

```text
rankTitle(rank, suffix)

rankTitle(1, "哥") = 大哥
rankTitle(2, "哥") = 二哥
rankTitle(6, "哥") = 六哥
rankTitle(4, "嫂") = 四嫂
rankTitle(11, "姐") = 十一姐
```

注意：

- 对“哥哥/姐姐”这类比我大的同辈，排行由同辈出生时间自动排序生成，比如大哥、二姐、三哥。
- 对“弟弟/妹妹”这类比我小的同辈，也由同辈出生时间自动排序生成二弟、三妹；如果业务希望从我之后重新计数，也需要单独配置。
- 对“嫂子”应使用其配偶，也就是哥哥的排行生成：大哥的妻子为大嫂，二哥的妻子为二嫂。
- 对“伯、舅、姨、姑”等长辈排行，应在父母那一代的同胞中排序。

## 称谓推导优先级

| 优先级 | 规则 |
| --- | --- |
| 1 | 先判断是否直系血亲，直系优先级最高 |
| 2 | 再判断是否配偶 |
| 3 | 再判断父系/母系旁系，决定伯叔姑舅姨 |
| 4 | 再判断同辈血缘，决定亲兄弟姐妹、堂亲、表亲 |
| 5 | 再判断姻亲，决定嫂子、弟媳、姐夫、妹夫、舅子、姨子等 |
| 6 | 最后叠加排行，大/二/三/四/五/六等，排行数量动态生成 |
